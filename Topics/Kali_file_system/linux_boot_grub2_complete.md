## 1. Kernel Images & System Maps

| **Category** | **Files** | **Explanation** |
|---|---|---|
| **Linux Kernel Image (6.12.25)** | `vmlinuz-6.12.25-amd64` | Compressed Linux kernel binary for x86_64 architecture. This is the core OS kernel that gets loaded by GRUB2 and decompressed into memory during boot. Contains all kernel code and built-in drivers |
| **Linux Kernel Image (6.16.8+kali)** | `vmlinuz-6.16.8+kali-amd64` | Kali-patched Linux kernel version 6.16.8. Includes security research patches for penetration testing tools, WiFi drivers, and vulnerability assessment features specific to Kali Linux |
| **System Map (6.12.25)** | `System.map-6.12.25-amd64` | Symbol map file for kernel 6.12.25 containing kernel symbol addresses. Used for debugging, kernel profiling, and /proc/kallsyms generation. Essential for kernel developers and crash debugging |
| **System Map (6.16.8+kali)** | `System.map-6.16.8+kali-amd64` | Symbol map for Kali kernel 6.16.8. Maps virtual memory addresses to kernel function and variable names for debuggers (GDB, kdb) and performance analysis tools |

## 2. Initial RAM Disk (Initrd)

| **Category** | **Files** | **Explanation** |
|---|---|---|
| **Initrd Image (6.12.25)** | `initrd.img-6.12.25-amd64` | Initial RAM filesystem for kernel 6.12.25. Compressed cpio archive containing minimal filesystem, device nodes, mount scripts, and storage drivers (SATA, NVMe, USB) needed BEFORE the real root filesystem is mounted. Essential for finding and mounting the actual root disk during boot |
| **Initrd Image (6.16.8+kali)** | `initrd.img-6.16.8+kali-amd64` | Initial RAM filesystem for Kali kernel 6.16.8. Contains hardware detection scripts and drivers for Kali-specific hardware (WiFi adapters, security dongles, penetration testing interfaces). Typically 50-100MB compressed |

## 4. GRUB2 EFI Bootloader

| **Category** | **Files** | **Explanation** |
|---|---|---|
| **GRUB2 EFI Binary (Kali)** | `/efi/EFI/kali/grubx64.efi` | UEFI bootloader executable loaded directly by firmware. This is the primary bootloader that runs on x86_64 UEFI systems. Located in EFI System Partition (ESP), loaded by firmware before any OS code executes |
| **GRUB2 EFI Core Module** | `/grub/x86_64-efi/core.efi` | Core EFI module containing GRUB2 runtime. Compiled for x86_64 UEFI systems with all essential functionality linked in. Provides menu, configuration loading, and kernel launch capabilities |
| **GRUB2 EFI Executable** | `/grub/x86_64-efi/grub.efi` | Secondary GRUB2 EFI executable for chain-loading. Can be used to boot from USB or create rescue boot media |

## 5. GRUB2 Configuration

| **Category** | **Files** | **Explanation** |
|---|---|---|
| **Main GRUB Config** | `grub/grub.cfg` | Primary GRUB2 configuration file. Contains boot menu entries, default timeout, resolution settings, and kernel parameters. Usually auto-generated by `update-grub` script from `/etc/grub.d/` templates. Syntax: BASH-like scripting language |
| **GRUB Environment File** | `grub/grubenv` | Persistent GRUB2 environment variables (100-1024 bytes). Stores default boot entry, theme preferences, and saved states between boots. Read/written by GRUB2 to persist user choices |

## Boot Process Flow

```
UEFI Firmware
    ↓
/efi/EFI/kali/grubx64.efi (GRUB2 EFI binary)
    ↓
grub/x86_64-efi/core.efi (GRUB2 core)
    ↓
Load grub/grub.cfg configuration
    ↓
Present boot menu (using theme/ graphics)
    ↓
User selects kernel entry
    ↓
Load kernel: vmlinuz-6.16.8+kali-amd64
    ↓
Load initramfs: initrd.img-6.16.8+kali-amd64
    ↓
Read System.map-6.16.8+kali-amd64 for kernel symbols
    ↓
Boot kernel with config-6.16.8+kali-amd64 parameters
    ↓
Kernel executes, mounts root filesystem
```

## Key Technical Details

### **Why Multiple Kernel Versions?**
- **6.12.25-amd64**: Stable/upstream Linux kernel
- **6.16.8+kali-amd64**: Kali-patched kernel with security research enhancements

### **Why Both initrd AND System.map?**
- **initrd.img**: Bootstrap the system before kernel can mount real root FS
- **System.map**: Maps kernel memory addresses to function names for debugging crashes

### **What Are GRUB2 Modules?**
- **Dynamic plugins** that GRUB2 loads as needed
- Enables support for any filesystem, encryption, video type, or bootloader
- Modular design allows small core (core.efi) with optional features

### **How Does GRUB2 Boot Multiple Kernels?**
- **grub.cfg** contains menu entries pointing to different kernels
- Each entry specifies:
  - Kernel image path (vmlinuz-X)
  - Initrd path (initrd.img-X)
  - Kernel parameters and boot options
- User selects from boot menu; GRUB2 loads selected pair

### **Security Aspects**
- **LUKS Modules** (luks.mod, luks2.mod) enable encrypted root filesystem support
- **Signature modules** (pgp.mod) allow signed kernel verification
- **Password protection** available via password.mod
- **Trusted Platform Module** (tpm.mod) for secure boot

## Practical Usage Examples

### **View current kernel:**
```bash
cat /proc/cmdline              # Shows kernel parameters used at boot
uname -r                       # Displays running kernel version
```

### **List installed kernels:**
```bash
ls -lh /boot/vmlinuz-*
ls -lh /boot/initrd.img-*
```

### **View GRUB2 configuration:**
```bash
cat /boot/grub/grub.cfg
grub-editenv list              # Show saved GRUB2 environment
```

### **Rebuild GRUB2 after changes:**
```bash
sudo update-grub               # Regenerates grub.cfg from templates
sudo grub-install /dev/nvme0n1 # Install/update bootloader
```

### **Access kernel symbols for debugging:**
```bash
grep kernel_function /boot/System.map-$(uname -r)
gdb vmlinux                    # Use with System.map for debugging
```
